# SourcingOS V17.3 API Connector Roadmap

## Current connector families
GitHub, Stack Overflow, OpenAlex, NPI, ORCID, Semantic Scholar, arXiv, PubMed, Hugging Face, npm, PyPI.

## V17.3 added lanes
Kaggle, DEV Community, Docker Hub, crates.io, RubyGems, and Public Resume X-Ray.

## Resume database stance
There is no durable, legal, free resume database API that should be treated like a bulk candidate database. SourcingOS should use user-uploaded resumes/CSVs, manually reviewed public resume/CV X-Ray links, paid/permissioned resume databases where the user has access, and ATS/CRM exports supplied by the user. Do not scrape LinkedIn, Indeed, ClearanceJobs, Dice, or resume databases.
